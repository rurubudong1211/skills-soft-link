# Product

## Register

product

## Platform

web

## Users

首位且当前唯一用户是产品创建者本人：一名在 Windows 上管理 Agent Skills 的高级用户。高保真原型以浏览器为交互载体，但产品体验面向 Windows 桌面工具。

## Product Purpose

管理多个源目录，展示其中全部一级文件和一级目录，把可选择的一级目录通过软链接分发到任意本机目标目录，并清楚查看、刷新和移除每个源子目录现有的软链接关系；Windows MVP 底层使用目录 Junction，因此一级文件只展示、不分发。原型的成功标准是能够完整体验 MVP 流程并发现交互问题，而不真实修改文件系统。

## Positioning

一份真实 Skill，清楚分发到哪里。

## Brand Personality

精确、克制、高效。界面应像可信赖的本地系统工具：路径和状态清楚，操作反馈直接，视觉明亮而安静。

## Anti-references

不采用黑底霓虹、命令行或黑客工具风格；不把简单文件操作包装成满屏统计卡片的 SaaS 仪表盘；不使用玻璃模糊、夸张圆角或无意义动画。

## Design Principles

- 以当前磁盘事实为准，不制造隐藏的同步状态。
- 让真实目录、软链接和仅用于应用记忆的路径在语言与操作上保持清晰边界。
- 路径、连接关系和冲突原因始终比装饰更醒目。
- 批量操作先预检、按项目反馈，允许部分成功且不回滚成功项。
- 外部变化由用户主动刷新，避免不可预测的后台扫描。

## Accessibility & Inclusion

达到 WCAG AA 对比度；核心操作可通过键盘完成；焦点状态清楚；状态不只依赖颜色；尊重系统的减少动态效果设置。原型使用语义化控件，但 MVP 不做额外的屏幕阅读器深度定制。
