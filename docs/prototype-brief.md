# Skills 软链接管理器高保真原型简报

> 状态：待确认。确认前不进入原型实现。

## 1. Feature Summary

为单一 Windows 高级用户设计的本地 Agent Skills 软链接管理工具。应用持久保存多个源目录，并通过已知目标路径创建、查看和移除目录软链接；Windows MVP 底层使用目录 Junction。交互式 Web 原型使用模拟数据验证完整 MVP 流程，不真实操作文件系统。

## 2. Primary User Action

用户应能快速选中一个或多个源子目录，确认它们将被分发到哪个目标目录，并在操作前看懂将创建、跳过或冲突的项目。

## 3. Design Direction

- **颜色策略**：克制型；纯白与中性浅灰承担结构，低饱和铜绿青只用于主操作、选中与焦点，语义色有限使用。
- **场景句**：用户在明亮的桌面环境中专注整理大量 Skills 与长路径，需要一个安静、可信、不会抢夺注意力的 Windows 工具。
- **参考锚点**：Microsoft PowerToys 的工具可信感、VS Code Explorer 的目录密度、GitHub Desktop 的本地状态表达。
- **反向约束**：不做黑客终端风格、SaaS 统计卡片、玻璃模糊、夸张圆角或无意义动画。
- **视觉探针**：内置图片生成服务因网络错误不可用，本轮不以生成图作为方向依据。

## 4. Scope

- **保真度**：高保真。
- **广度**：覆盖完整 MVP 表面与关键状态。
- **交互性**：可点击的浏览器原型；使用模拟数据，不接触真实文件系统。
- **验证目标**：让用户实际走完流程，发现交互问题并提出修改。
- **画布**：主要尺寸 `1280 × 800`，支持至约 `1024 × 680`；更窄时右侧详情改为抽屉，不设计移动端。

## 5. Layout Strategy

主界面采用三栏桌面结构：左侧是多个源目录；中间是当前来源的全部一级文件和一级目录；右侧是当前聚焦条目的详情。左栏负责切换上下文，中栏承担搜索、筛选、多选与批量分发，右栏承担查看文件说明或目录的约十条目标路径并逐条移除软链接。

中栏保持约 `40px` 行高，在普通 1080p 窗口中同时展示约 18–22 项。一级内容按名称自然升序排列；列表提供「全部 / 未连接 / 已连接」筛选：「全部」展示所有文件和目录，「未连接」展示所有文件及未连接目录，「已连接」只展示已连接目录。搜索位于列表上方，「全选」只选择当前结果中的目录，文件不可勾选。勾选负责批量操作，单击行负责聚焦详情，两种状态在视觉上明确分离。

没有独立目标目录管理页。目标路径只在分发选择面板的最近使用区和右侧连接详情中出现。

## 6. Key States

- **首次使用**：没有源目录，解释单一真实来源模型并提供「添加源目录」。
- **正常列表**：显示 20–80 个常见文件和目录，并能承载最多约 300 个一级条目。
- **空源目录**：目录可访问但没有一级文件或目录，保留添加来源和刷新入口。
- **源目录不可用**：来源被删除、改名或暂时无法访问；不在源目录列表中展示，也不弹全局错误，恢复可访问后重新显示。
- **加载与刷新**：首次打开或切换来源自动扫描；之后只由「刷新」触发，列表使用结构化骨架状态。
- **未选择 / 已多选**：未选择时主操作不可用；选择后出现明确的已选数量与「分发到…」。
- **目标选择**：展示已知目标路径和「浏览新目录」；不可用路径可被忘记，但不修改磁盘。
- **预检**：汇总「将创建 / 已连接 / 名称冲突」，并逐项标识；已连接和冲突项跳过。
- **执行结果**：允许部分成功；成功项保留，失败项显示具体原因，不回滚、不弹全局错误框。
- **连接详情**：通常展示约 10 个目标父目录路径；路径支持截断与完整提示，可逐条「移除软链接」。
- **外部删除**：用户在应用外移除软链接后，下一次手动刷新让关系自然消失，不报错。

## 7. Interaction Model

1. 用户从左栏选择或添加源目录；切换时自动读取其全部一级文件、一级目录和已知目标中的当前关系。
2. 用户通过名称搜索与连接状态筛选缩小列表，再勾选多个目录或全选当前结果中的目录；文件可在「全部」和「未连接」中查看，但不可勾选。
3. 点击「分发到…」打开轻量目标选择面板；复用最近目标或浏览新目录，新目录自动成为已知目标路径。
4. 选定目标后在同一面板显示预检摘要与项目状态；点击一次「创建软链接」执行模拟操作。
5. 完成后在列表和结果区更新连接数量；部分失败仅影响对应项目。
6. 单击源子目录行，在右侧查看所有软链接所在父目录；点击「移除软链接」直接移除该链接，不显示确认，不影响源目录。
7. 从源目录列表移除一项只让应用忘记来源，不删除来源或已有软链接；从目标选择面板忘记路径同样不修改磁盘。

常规状态变化使用 `150–200ms` 的低能量淡入或短位移，并为减少动态效果提供即时替代。核心交互具有键盘焦点、可预测的 Tab 顺序和清楚的选中状态。

## 8. Content Requirements

- 源目录名称与完整路径、可用状态、一级内容数量。
- 一级文件和目录的名称、类型与聚焦状态；目录额外显示连接数量和选中状态。
- 目标父目录完整路径、可用状态和逐条移除动作。
- 搜索占位文案、连接状态筛选、已选数量、刷新时间或刷新状态。
- 预检汇总与逐项状态：「将创建」「已连接」「名称冲突」。
- 执行结果：「创建成功」「已跳过」「创建失败」及简短可行动原因。
- 空状态、不可用状态和部分失败状态均使用简短直白的中文文案，不使用技术炫耀式语气。

原型使用可信的 Windows 示例路径与 Agent Skill 名称，不需要插画、照片或装饰图形；图标使用一致的线性系统图标。
