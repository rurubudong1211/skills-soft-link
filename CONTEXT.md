# Skills 软链接管理器

一个展示多个源目录的一级内容、把其中的源子目录分发到多个本机目录，并追踪其软链接关系的个人管理工具。

## Language

**源目录**：
由用户登记并持久保存的真实目录根节点；应用读取其全部一级文件和一级目录。应用可维护多个源目录。
_避免：技能源、技能源目录、Agent 目录_

源目录被删除、改名或暂时无法访问时，应用可保留其内部登记记录，但不在源目录列表中展示，也不弹出错误；目录恢复可访问后重新显示。

**当前源目录**：
用户当前在源目录列表中选中的一项；主区域展示该目录的全部源条目。
_避免：默认源目录_

**移除源目录登记**：
仅从应用的源目录列表中移除一项；源目录、真实子目录和现有软链接全部保持原样。
_避免：删除源目录、清理软链接_

**源条目**：
源目录下的任意一级文件或一级目录，均在主列表中展示。
_避免：只用源子目录指代整个列表_

**源子目录**：
源条目中的一级目录，不要求包含 `SKILL.md`，也不做其他内容或结构校验；只有源子目录可以被选择并创建软链接。一级文件仅展示，不可选择或分发。
_避免：有效 Skill、已验证 Skill_

源条目列表提供「全部 / 未连接 / 已连接」筛选。「全部」显示所有一级文件和目录；「未连接」显示全部一级文件及尚无软链接的目录；「已连接」只显示至少存在一个当前软链接关系的目录。列表可按名称搜索，「全选」只选择当前结果中可分发的目录，不选择文件。

源条目固定按名称自然升序展示，例如 `skill-2` 位于 `skill-10` 之前；不使用文件系统枚举顺序。

**软链接**：
界面中对受管理目录链接的统一称呼；Windows MVP 底层使用目录 Junction。它位于目标目录中，名称与源子目录相同并指向该源子目录；本产品不管理别名、文件链接或其他链接类型。
_避免：在界面中使用 Junction、文件软链接、复制目录、软链接别名_

**目标目录**：
用户选择的任意本机目录，应用可在其中为技能创建软链接；它不代表某个 Agent，也不要求固定用途。
_避免：Agent 技能目录、分发目录_

**已知目标目录**：
应用为反向扫描而在内部全局记住的目标路径，不隶属于某个源目录，也没有独立管理页面。它只在分发的目标选择面板和源子目录的连接详情中按上下文出现；应用只扫描这些路径来发现软链接关系。
_避免：目标目录管理列表、全盘搜索范围_

**忘记目标路径**：
仅从应用内部的已知目标目录中移除一项；目标目录及其中的普通目录、软链接全部保持原样。该动作只出现在分发目标选择面板的最近使用区域。
_避免：删除目标目录、清理软链接_

已知目标目录被删除、改名或暂时无法访问时，应用保留其记录并标记为「不可用」，跳过扫描且不弹出错误；用户可手动移除记录，目录恢复后可刷新识别。

**分发操作**：
用户选择一个或多个源子目录，并在一次操作中为它们创建指向同一个目标目录的软链接；用户可复用已知目标目录或浏览新目录，新目录会自动成为已知目标目录。用户可通过多次操作分发到不同目标目录。
_避免：同步、复制_

若目标目录中已存在指向同一源子目录的同名软链接，该源子目录显示为「已连接」并跳过创建。

若目标目录中存在同名普通文件夹，或存在指向其他位置的同名软链接，该源子目录显示为「名称冲突」并跳过；同一分发操作中的其他无冲突项继续创建。

分发操作允许部分成功。单项失败不回滚已成功创建的软链接，也不阻止其他项目继续；操作结果按源子目录分别展示。

**移除软链接**：
仅删除目标目录中的软链接入口，不提示确认，也不删除或修改源子目录；产品不提供删除源子目录的能力。
_避免：删除目录、删除 Skill_

用户可在应用外移除软链接。刷新时，已不存在的关系直接消失，不视为错误，也不产生告警。

应用不保存软链接关系的历史快照；当前可访问目录中的磁盘事实是唯一依据。重新登记相同路径后，现有关系通过扫描自动恢复显示。

应用不监听后台文件变化，也不在窗口重新获得焦点时自动扫描。外部变化仅在用户主动点击「刷新」后同步到当前界面。

首次打开应用或切换当前源目录时，应用自动扫描一次；本次加载完成后的重新扫描仍由「刷新」触发。
