# Design System: Skills 软链接管理器

## Overview

**Creative North Star: “明亮的连接工作台”**

界面像一张保持整洁的本地工作台：路径、连接关系和下一步动作始终位于视线中心。整体借鉴 Airbnb 的友好留白、圆润搜索与单一品牌强调色，同时保留 VS Code Explorer 的目录密度，以及 GitHub Desktop 对复杂本地状态的平静表达。

明亮主题服务于长时间查看路径与列表；动效只解释状态变化，绝不编排表演。主画布面向 `1280 × 800`，在约 `1024 × 680` 时把右侧连接详情收为抽屉，不设计手机端。

**Key Characteristics:**

- 精确、克制、高效
- 明亮的中性表面与少量珊瑚红强调
- 偏紧凑的桌面信息密度
- 路径与状态优先于装饰
- 150–200ms 的低能量状态动效

## Colors

采用克制型颜色策略：纯白背景、中性浅灰分层，以及使用面积不超过约 10% 的珊瑚红主强调色。成功、冲突和不可用继续使用独立语义色，所有状态同时配合文字或图标。

### Primary

- **克制珊瑚红**（待原型阶段解析）：参考 Airbnb Rausch，但适度压低明度以满足小字号白字的 WCAG AA 对比度；用于主操作、当前选择和键盘焦点，不用于大面积装饰。

### Neutral

- **工作台白**（待原型阶段解析）：应用主背景。
- **工具面灰**（待原型阶段解析）：侧栏、工具栏和详情区域的层级表面。
- **路径墨色**（待原型阶段解析）：正文、路径和关键状态文字，必须达到高可读对比度。

**The Rare Accent Rule.** 珊瑚红只标识动作、选中与焦点；如果第一眼看到的是颜色而不是路径，强调色就用多了。

## Typography

**Display Font:** 单一 Windows 系统无衬线方向；原型优先采用 Segoe UI Variable。
**Body Font:** 与界面统一的 Windows 系统无衬线方向。

**Character:** 熟悉、稳定、紧凑。路径同样使用系统字体，通过字号、字重、截断和悬停完整提示建立层级，不用等宽字体制造终端感。

### Hierarchy

- **Headline**（待原型阶段解析）：页面与当前源目录标题。
- **Title**（待原型阶段解析）：面板标题与关键操作名称。
- **Body**（待原型阶段解析）：目录名、路径与操作反馈。
- **Label**（待原型阶段解析）：筛选、状态和辅助元数据。

**The One Family Rule.** 产品界面只使用一套系统无衬线字体，不引入展示字体或等宽字体。

## Elevation

默认采用平面和色调分层；侧栏、内容区与详情区通过背景差异和细分隔线建立结构。阴影只允许用于临时浮层，例如目标选择面板，并保持短而清晰。

**The Flat Workbench Rule.** 常驻界面不依靠宽大柔影制造卡片；浮层关闭后，阴影也随之消失。

## Do's and Don'ts

### Do:

- **Do** 让长路径、连接数量和操作结果在紧凑列表中仍可快速扫描。
- **Do** 使用珊瑚红突出主操作、当前选择和焦点，并将其限制在小面积范围。
- **Do** 使用 150–200ms 的状态动效，并提供减少动态效果替代。
- **Do** 在普通 `1080p` 窗口中同时展示约 18–22 个源子目录。

### Don't:

- **Don't** 采用黑底霓虹、命令行或黑客工具风格。
- **Don't** 把简单文件操作包装成满屏统计卡片的 SaaS 仪表盘。
- **Don't** 使用玻璃模糊、夸张圆角或无意义动画。
- **Don't** 仅依赖颜色表达已连接、冲突、不可用或失败状态。
